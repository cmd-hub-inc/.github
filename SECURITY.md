# Security Policy

## Reporting a Vulnerability

If you believe you have found a security vulnerability in this project, please report it to the maintainers immediately by emailing security@example.com. Replace this address with a project-specific contact or a vulnerability disclosure platform. If you prefer encrypted reports, include our PGP key: <PGP_KEY_PLACEHOLDER>.

When reporting, include:
- A clear description of the issue and steps to reproduce it.
- Proof-of-concept code or a minimal test case if available.
- Your contact information and an indication whether you consent to being credited.

## Response Process

We aim to acknowledge all reports within 3 business days and will work to provide a remediation or mitigation timeline based on severity. We follow coordinated disclosure: we will not publicly disclose a vulnerability until a fix or mitigation is available.

## Supported Versions

We provide fixes for the current release branch and the previous major version. If a vulnerability affects an unsupported version, we will evaluate whether a patch is feasible.

## Severity & CVE

We will assign severity based on impact and likelihood. For issues that meet criteria, we will request a CVE and coordinate publication after a patch is released.

## Safe Harbor

We welcome good-faith security research. Reporters who act in good faith and follow this policy will not be subject to legal action by the maintainers.

## Credit

With your permission, we will credit security researchers who report verified vulnerabilities in release notes or the `SECURITY.md` file.
